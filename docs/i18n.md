# Internationalization (i18n)

**Text domain:** `prikogstreg-online-invitations`  
**Domain path:** `/languages`  
**Source language:** English (strings in PHP/JS use WordPress i18n APIs)  
**Target locale (V1):** Danish (`da_DK`)

---

## Rules

1. All user-facing PHP strings use `__()`, `_e()`, `esc_html__()`, `esc_attr__()`, or `printf()` with translated format strings.
2. JavaScript UX strings are passed from PHP via `wp_localize_script` — no hard-coded production copy in bundled JS except fallbacks.
3. Do not load Node-based i18n tooling at runtime.

---

## Generate POT template

From the plugin root (requires [WP-CLI](https://wp-cli.org/) i18n commands):

```bash
wp i18n make-pot . languages/prikogstreg-online-invitations.pot \
  --domain=prikogstreg-online-invitations \
  --exclude=vendor,node_modules,pdf-plugin,assets/src
```

Update the `.pot` after adding or changing translatable strings.

---

## Danish translations

Shipped file: `languages/prikogstreg-online-invitations-da_DK.po`

Compile to MO for WordPress (optional local step):

```bash
wp i18n make-mo languages/
```

Or use Poedit / `msgfmt`:

```bash
msgfmt -o languages/prikogstreg-online-invitations-da_DK.mo languages/prikogstreg-online-invitations-da_DK.po
```

WordPress loads translations when `load_plugin_textdomain()` runs (see main plugin bootstrap).

---

## Theme overrides

Templates remain translatable in PHP. Theme overrides copy plugin templates to:

```text
{child-theme}/prikogstreg-online-invitations/{template}.php
```

See `docs/theme-overrides.md` for resolution order and `@version` compatibility headers.
