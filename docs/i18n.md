# Internationalization (i18n)

**Text domain:** `prikogstreg-online-invitations`  
**Domain path:** `/languages`  
**Source language:** English (strings in PHP/JS use WordPress i18n APIs)  
**Target locale (V1):** Danish (`da_DK`)

---

## Rules

1. All user-facing PHP strings use `__()`, `_e()`, `esc_html__()`, `esc_attr__()`, or `printf()` with translated format strings.
2. JavaScript UX strings are passed from PHP via `wp_localize_script` — no hard-coded production copy in bundled JS except fallbacks.
3. Do not load Node-based i18n tooling at runtime.

---

## Full build (POT + Danish PO/MO)

```bash
npm run i18n
```

This runs `scripts/i18n.sh`:

1. `wp i18n make-pot` → `languages/prikogstreg-online-invitations.pot`
2. `python3 scripts/build-da-po.py` → updates `languages/prikogstreg-online-invitations-da_DK.po` and compiles `.mo`

Regenerate the POT only:

```bash
npm run i18n:pot
```

Requires [WP-CLI](https://wp-cli.org/) and Python 3 with `polib` (`pip3 install polib`).

---

## Generate POT template (manual)

From the plugin root:

```bash
wp i18n make-pot . languages/prikogstreg-online-invitations.pot \
  --domain=prikogstreg-online-invitations \
  --exclude=vendor,node_modules,pdf-plugin,assets/src
```

Update the `.pot` after adding or changing translatable strings, then run `npm run i18n` to refresh Danish files.

---

## Danish translations

Shipped files:

- `languages/prikogstreg-online-invitations-da_DK.po` — editable catalog
- `languages/prikogstreg-online-invitations-da_DK.mo` — compiled for WordPress

Edit Danish strings in `scripts/build-da-po.py` (`TRANSLATIONS` / `PLURAL_TRANSLATIONS`), then run `npm run i18n`.

WordPress loads translations when the site locale is `da_DK` and `load_plugin_textdomain()` runs in `Plugin::boot()`. English `msgid` strings in source code are always the default fallback.

## Theme overrides

Templates remain translatable in PHP. Theme overrides copy plugin templates to:

```text
{child-theme}/prikogstreg-online-invitations/{template}.php
```

See `docs/theme-overrides.md` for resolution order and `@version` compatibility headers.
